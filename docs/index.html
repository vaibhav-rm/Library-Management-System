Introduction

Welcome to the documentation for our Learning Management System (LMS) project. This guide will help you understand the features and functionality of our LMS, as well as how to set it up and use it effectively.

Getting Started

To get started with our LMS, follow these steps:

Installation

git clone https://github.com/vaibhav-rm/Learning-Managemet-System.git
cd Learning-Managemet-System
npm install

Install backend dependencies

cd backend 
npm install

Configuration

  1. Set up your environment variables, use .env.sample file
    2. PORT=backend port
MONGODB_URI=mongodb url
CORS_ORIGIN=frontend url
ACCESS_TOKEN_SECRET=access token string
ACCESS_TOKEN_EXPIRY=access token expiry time
REFRESH_TOKEN_SECRET=refresh token string
REFRESH_TOKEN_EXPIRY=refresh token expiry time

CLOUDINARY_CLOUD_NAME=cloudinary username
CLOUDINARY_API_KEY=cloudinary key
CLOUDINARY_API_SECRET=cloudinary secret
  2. Run the development server: npm run dev
  3. Run the backend server: cd backend, npm run dev

API Reference

Overview

The LMS API provides programmatic access to user, book, and author management functionalities.

Base URL

All API requests should be made to: http://localhost:8000/api/v1

Authentication

Most endpoints require JWT authentication. Include the JWT token in the Authorization header:

Authorization: Bearer <your_jwt_token>

Standard Responses

The API uses standardized response and error formats, implemented through ApiResponse and ApiError classes.

    "message": "Error message",
    "errors": [

Standard API Responses

This API uses standardized response and error formats to ensure consistency across all endpoints.

Success Responses

All successful API responses follow this structure:

{
    "statusCode": number,
    "data": object | null,
    "message": string,
    "success": boolean
}

Example of a successful response:

{
    "statusCode": 200,
    "data": {
        "user": {
            "id": "123",
            "username": "johndoe"
        }
    },
    "message": "User retrieved successfully",
    "success": true
}

Error Responses

All error responses follow this structure:

{
    "statusCode": number,
    "data": null,
    "message": string,
    "success": false,
    "errors": array
}

Example of an error response:

{
    "statusCode": 400,
    "data": null,
    "message": "Validation error",
    "success": false,
    "errors": [
        "Username is required",
        "Email must be a valid email address"
    ]
}

Implementation Details

The API uses two utility classes to generate these standardized responses:

ApiResponse Class

Used for successful responses:

class ApiResponse {
    constructor(statusCode, data, message = "Success"){
        this.statusCode = statusCode
        this.data = data
        this.message = message
        this.success = statusCode < 400 
    }
}

ApiError Class

Used for error responses:

class ApiError extends Error {
    constructor(
        statusCode,
        message = "Something went wrong",
        errors = [],
        stack = ""
    ){
        super(message)
        this.statusCode = statusCode
        this.data = null
        this.message = message
        this.success = false
        this.errors = errors

        if (stack) {
            this.stack = stack
        } else {
            Error.captureStackTrace(this, this.constructor)
        }
    }
}

Usage Examples

Here are some examples of how to use these classes in your API controllers:

Successful Response Example

import { ApiResponse } from '../utils/ApiResponse.js';

const getUserProfile = async (req, res) => {
    try {
        const user = await User.findById(req.user.id);
        
        if (!user) {
            throw new ApiError(404, "User not found");
        }

        const response = new ApiResponse(200, { user }, "User profile retrieved successfully");
        res.status(200).json(response);
    } catch (error) {
        next(error);
    }
};

Error Handling Example

import { ApiError } from '../utils/ApiError.js';

const createUser = async (req, res, next) => {

        }

        const existingUser = await User.findOne({ email });
        if (existingUser) {
            throw new ApiError(409, "User with this email already exists");
        }

        // ... create user logic ...

        const response = new ApiResponse(201, { user: newUser }, "User created successfully");
        res.status(201).json(response);

Global Error Handling Middleware

You can use a global error handling middleware to catch and format all errors:

import { ApiError } from '../utils/ApiError.js';

const errorHandler = (err, req, res, next) => {
    if (err instanceof ApiError) {
        return res.status(err.statusCode).json({
            statusCode: err.statusCode,
            data: err.data,
            message: err.message,
            success: false,
            errors: err.errors
        });
    }

    // For unhandled errors
    const response = new ApiError(500, "Something went wrong", []);
    res.status(500).json(response);
};

export default errorHandler;

By using these standardized classes and a global error handler, you ensure consistent response formats across your entire API. This makes it easier for frontend developers to handle responses and errors in a uniform way.
